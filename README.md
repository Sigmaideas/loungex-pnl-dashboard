# 라운지엑스24h P&L 대시보드

라운지엑스24h 무인 로봇 매장의 **지점별 손익(P&L)**을 한 화면에서 보고 입력·수정할 수 있는 정적 대시보드입니다. 백엔드 없이 브라우저 `localStorage`에 데이터가 저장되며, JSON 파일로 내보내기/불러오기를 지원합니다.

## 1. 프로젝트 소개

- 매장별 매출, 투자금, 회수금, 회사 P&L을 한 화면에서 확인
- 기간 필터 + 4개 KPI 카드 + 매출 비중 도넛 차트 + 지점 상세 테이블 + 월별 실적 입력
- 모든 셀은 **더블클릭으로 편집** (Enter 또는 포커스 이동으로 저장, Esc로 취소)
- 데이터는 자동으로 `localStorage`에 저장되며, 화면을 닫았다가 다시 열어도 유지됩니다
- 단일 페이지 / 단일 화면 / 다크 모드 / 한국어 UI

## 2. 로컬 실행

별도 빌드 도구가 필요 없습니다. 디렉토리에서 다음 중 하나를 실행하세요.

```bash
# Python 3
python3 -m http.server 8000

# 또는 Node가 있다면
npx serve .
```

브라우저에서 <http://localhost:8000> 접속.

> `index.html`을 더블클릭해 `file://` 로 열어도 동작하지만, 일부 브라우저에서 `localStorage` 키 격리 등의 동작이 다를 수 있으니 로컬 서버 사용을 권장합니다.

## 3. 사용법

### 3.1 데이터 입력/수정
- **모든 편집은 더블클릭**: 지점명, 오픈일, 투자금, 오픈수익, 운영수익률, 월별 매출, 투자자 회수금 모두 셀을 더블클릭하면 편집창이 뜹니다.
- Enter 또는 다른 곳을 클릭(blur)하면 저장됩니다. Esc로 취소.
- **운영일자**(오픈일 → 오늘)와 **운영수익**, **수익률**은 자동 계산되어 편집할 수 없습니다.

### 3.2 지점 추가
- "지점 상세" 카드 우상단의 **"+ 지점 추가"** 버튼을 누르면 기본값으로 새 지점이 생성됩니다.
- 생성된 행의 각 셀을 더블클릭해 값을 채워주세요.

### 3.3 월별 실적 입력
- "월별 실적 입력" 카드의 좌측 드롭다운에서 지점을 선택합니다.
- **"+ 월 추가"** 버튼을 누르면 마지막 월 다음 달이 자동으로 추가됩니다.
- 매출과 투자자 회수금을 더블클릭해 입력하면 운영수익이 자동 갱신됩니다.

### 3.4 기간 필터
- 화면 상단의 시작/종료 연월로 KPI, 차트, 표의 합계 계산 범위를 바꿀 수 있습니다.
- "전체 기간", "최근 12개월" 빠른 선택 버튼 제공.

### 3.5 데이터 초기화
- **데이터 초기화**: 모든 데이터를 지우고 샘플 데이터로 되돌립니다.

## 4. P&L 계산 공식

| 항목 | 공식 |
|---|---|
| 운영일자 | `오늘 - 오픈일` (일 단위) |
| 운영수익 (월) | `매출 × 운영수익률` |
| 회사 P&L (선택 기간) | `해당 기간 내 오픈 시 오픈수익(1회) + Σ 월별 운영수익` |
| 최소 회수금액 (월) | `총 투자금액 / 60` |
| 평균 매출 (월) | `Σ 매출 / 입력된 월수` |
| 평균 회수금액 (월) | `평균 매출 (월) × 20%` |
| 수익률 | `평균 회수금액 (월) / 최소 회수금액 (월)` |
| 평균 수익률 (KPI) | 모든 지점의 `수익률` 평균 |

> **참고**
> - 오픈수익은 오픈한 달이 선택 기간에 포함될 때만 1회 계산됩니다.
> - "총 회수금액"은 기간 필터와 무관하게 누적 회수금을 표시합니다(투자자 관점). KPI 카드의 합계에서도 이 값을 그대로 사용합니다.
> - 운영수익률은 지점별로 개별 설정할 수 있으며, 기본값은 20% 입니다.

## 5. GitHub Pages 배포

1. GitHub에 저장소 생성 후 푸시:
   ```bash
   git init
   git add .
   git commit -m "init: loungex pnl dashboard"
   git branch -M main
   git remote add origin https://github.com/<사용자명>/<저장소명>.git
   git push -u origin main
   ```
2. 저장소 **Settings → Pages**:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main` / `(root)`
3. 1~2분 후 `https://<사용자명>.github.io/<저장소명>/` 에서 접속 가능합니다.

> 정적 파일만 있으므로 별도 빌드 단계 없이 바로 배포됩니다.
